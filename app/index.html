Blocks

In SPAR, you track your mining in 'blocks'. Blocks represent arbitrary chunks of time in which you're mining. You can start and end these however you like, but to ensure that the report coming out of SPAR constitutes as a accurate representation of what you're doing in-game as possible, you should follow these guidelines:

  • When a new member joins you to start mining, end the current block. Start a new block and add them as a participant.
  • Categorize your participants.
  • Track the right ores using the filters on the right. You can remove ores that aren't present by clicking the 'x' that appears when you hover it.

Views

SPAR contains several views:

  • The Mining View, in which you start, populate, and end blocks. Once you end the session, you will pass automatically into the reporting view.
  • In theReporting View, you can view your results. If you find that you need to alter the data, you can go back to the mining view and make any necessary changes.
  • The Data Export View shows you a JSON version of the session -- basically, this is a dump of the model at the time of the session ending (including further edits up to the point of generation). Persistent storage via database, API stuff, login & history features may be added if the tool sees enough use for me to spend the time for it.
